# 数据库锁

#### 概念

- 并发控制的重要技术
- 事务对某个对象操作时候，先对其加锁。拥有其控制权

#### 分类

- **共享锁**（S Lock): 
  - 多个事务封锁一个共享页
  - 任何事务不能修改
  - 读取完毕释放锁
- **排它锁** （X Lock):
  - 仅运行一个事务封锁
  - 其它事务必须等待
  - 事务结束释放
- **更新锁** （U Lock):
  - 修改操作初始化阶段锁定要修改的资源
  - 避免获得两个共享锁想升拍它锁发生互锁（两个不互相释放共享锁）
- 意向锁 
  - 标志 说明某个资源已经被锁
